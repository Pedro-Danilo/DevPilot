Describir usuarios representativos para orientar UX, permisos, lenguaje, soporte y prioridades.

## 3. Ejemplo completo

| Campo | Contenido |
|---|---|
| Nombre | Marta, vendedora de emprendimiento local |
| Rol | Registra ventas y consulta inventario |
| Contexto | Atiende por WhatsApp e Instagram |
| Objetivos | Registrar ventas en menos de 30 segundos |
| Frustraciones | Olvida actualizar inventario |
| Nivel técnico | Bajo/medio |
| Dispositivos | Celular Android |
| Restricciones | Poco tiempo, conectividad irregular |
| Escenarios | Venta rápida, actualización de stock, consulta de cliente |
| Métricas | tiempo por venta, errores, recurrencia |

## 4. Criterios de revisión
- Representa decisiones de diseño.
- No es un perfil decorativo.
- Tiene contexto y restricciones reales.

## 5. Criterios de rechazo
- Persona ficticia sin relación con decisiones.
- No contiene frustraciones ni restricciones.
