# MVP Scope

## 1. Propósito
Definir el experimento mínimo funcional para validar hipótesis críticas sin inflar alcance.

## 2. Campos obligatorios

| Campo | Contenido |
|---|---|
| Hipótesis principal |  |
| Usuarios objetivo |  |
| Capacidades incluidas |  |
| Capacidades excluidas |  |
| Métricas de éxito |  |
| Duración de validación |  |
| Riesgos |  |
| Criterios de continuar |  |
| Criterios de pausa |  |
| Criterios de abandono |  |

## 3. Ejemplo completo

| Campo | Contenido |
|---|---|
| Hipótesis | Los usuarios registrarán ventas si el flujo toma menos de 30 segundos. |
| Usuarios | 3 microemprendimientos locales. |
| Incluido | productos, ventas, inventario simple, reporte básico. |
| Excluido | facturación electrónica, contabilidad formal, pagos automáticos complejos. |
| Métricas | ventas registradas, uso semanal, errores. |
| Duración | 30 días. |
| Continuar | 2/3 negocios usan el sistema semanalmente. |
| Pausar | dudas sobre flujo o valor. |
| Abandonar | nadie lo usa sin asistencia constante. |

## 4. Criterios de revisión
- Incluye exclusiones explícitas.
- Mide aprendizaje, no solo entrega.
- Tiene criterios de abandono.

## 5. Criterios de rechazo
- El MVP contiene todo el producto final.
- No hay hipótesis validable.
